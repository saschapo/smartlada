# Силовой ключ — модуль D4184 (low-side)

Каждый из 4 каналов SmartLada коммутируется покупным модулем **«High power MOS
Driver/Switch»** на сдвоенном N-канальном MOSFET **D4184** (low-side, current sink).
ESP32-C6 подаёт на вход `TRIG/PWM` сигнал LEDC, модуль ключует минус лампы на землю.

## Паспорт модуля

| Параметр | Значение |
|----------|----------|
| Транзисторы | 2× **D4184** (N-канал), включены **параллельно** |
| Тип ключа | **Low-side** (current sink), `+Vin` и `+Vout` соединены на плате |
| Питание нагрузки | 5–36 В DC |
| Ток | 15 А (400 Вт); до 30 А с радиатором |
| Уровень триггера | 3.3–20 В, **логический** (3.3 В ESP32-C6 открывает ключ) |
| Rds(on) | ~0.007 Ом |
| **Макс. частота PWM** | **20 кГц** *(см. предупреждение ниже)* |
| Рабочая t° | −40…+85 °C |

## Схема

**Входная (затворная) цепь.** Между `TRIG/PWM` и затвором — 100 Ω (ограничение тока
заряда затвора). На затворе — **100K pull-down** (держит ключ закрытым, пока вход не
подан) и индикатор **1.0K + LED** (горит, когда ключ открыт).

```
  ESP32-C6         100 Ω          gate_node ── затворы D4184 ×2
  GPIOx ──[TRIG]───[███]──────────┬──────────┬─────────────
 (LEDC PWM)                       │          │
                               [100K]     [1.0K]
                             pull-down       │
                                  │         LED (индикатор ON)
                                  │          │
  GND ────────────────────────────┴──────────┴──────────────  VIN− / GND
```

**Силовая цепь (low-side).** Плюс лампы сидит на общем +12 В (перемычка `+Vin = +Vout`
на плате), а минус лампы (`VOUT−`) коммутируется транзистором на землю:

```
                       +12 В  (Ecola B2L080ESB, DC · предохранитель T4A slow-blow)
                          │
          ┌───────────────┴───────────────┐
       VIN+                            VOUT+   ← перемычка на плате: +Vin = +Vout
          │                               │
      [ модуль ]                      ┌────┴────┐
          │                           │  R10W   │  лампа канала
          │                           │  12 В   │  (стоп / задний / поворот / габарит)
          │                           └────┬────┘
          │                                │ VOUT−  ← коммутируется минус лампы
          │                           drain │
          │                           ┌─────┴─────┐
          │                           │ D4184 ×2  │  N-канал, параллельно
          │                           └─────┬─────┘
          │                          source │
       VIN− ─────────────────────────────────┴──────  GND
          │
  GND ────┴────────────────────  общая земля с ESP32-C6 (опора для PWM, обязательна)
```

## Привязка к проекту

| Что | Значение | Источник |
|-----|----------|----------|
| Входы `TRIG/PWM` | `GPIO0..3` (CH0 стоп, CH1 задний, CH2 поворот, CH3 габарит) | `src/config/config.cpp` |
| Драйвер PWM | аппаратный LEDC, 10 бит | `src/platform_pwm/` |
| Частота по умолчанию | 30 000 Гц | `FREQ_DEFAULT` |
| Лампы | R10W 12 В BA15S, 40 Вт / 3.33 А суммарно | README |
| Источник +12 В | Ecola B2L080ESB (DC, 80 Вт, 6.6 А, защита КЗ) | README |

Почему low-side подходит: источник — стабилизированный DC-блок (не электронный
трансформатор), поэтому ключевание минуса корректно. Лампы **переполюсованы на общий
+12 В** — это и есть требование low-side-модуля (`+Vin = +Vout`).

## ⚠️ Предупреждения

- **Частота PWM выше паспортных 20 кГц — но при этой нагрузке это не проблема.**
  Дефолт прошивки — 30 кГц (он же потолок настройки), паспорт модуля — макс. 20 кГц.
  Ограничение паспорта связано с коммутационными потерями (слабая раскачка затвора: 100 Ω
  от 3.3 В, без gate-драйвера), но эти потери **линейны по току**, а ток здесь мал:
  R10W = ~0.83 А/канал, делится на два параллельных D4184 (~0.42 А на транзистор).
  Оценка потерь на канал при 30 кГц: проводимость ~0.003 Вт, коммутация ~0.15–0.45 Вт —
  т.е. ~0.1–0.2 Вт на транзистор, корпус рассеивает не нагреваясь. Подтверждено: **3 ч
  непрерывной работы на 30 кГц — не греются ни MOSFET, ни фара, ни ESP32, ни БП**
  (2026-07-11); тепловое равновесие достигается за минуты, так что режим установившийся.
  Номинал 20 кГц стал бы актуален только при существенном росте тока на канал (лампы
  мощнее, лента, много каналов на 100% одновременно) — тогда контролировать нагрев или
  снизить `pwm_freq_hz`.
- **Затворный pull-down обязателен** и уже есть на модуле (100K). Он держит ключ
  закрытым, пока `GPIO0..3` в boot не сконфигурированы. На первом включении проверить
  «duty 0 = лампа выключена». В прошивке дублируется `forceAllLow()` до любой инициализации.
- **Общая земля** ESP32-C6 ↔ GND БП — опора для PWM, без неё ключ работает нестабильно.
- **Предохранитель** по +12 В — **T4A slow-blow** (номинал 3.33 А, терпит soft-start
  бросок холодной нити ~8–10 А/лампу; `soft_start_ms=0` с этими лампами не ставить).
