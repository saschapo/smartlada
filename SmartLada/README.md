# SmartLada — калибровочный стенд PWM (ESP8266)

Bring-up «железа» умного светильника из фонаря ВАЗ-2106: 4 лампы накаливания 12 В
через 4 модуля D4184 (N-MOSFET, low-side PWM). **Это НЕ финальная прошивка** — стенд
для снятия чисел калибровки, пока едет ESP32-C6. Финал — Zigbee на C6.

> **Статус: прототип рабочий (2026-07-06).** Схема подключения ниже проверена на
> реальных лампах фонаря: 4 канала диммируются независимо, мерцания нет.
> Снятая калибровка — в [calibration/](calibration/).

## ⚠️ Силовой стенд — прочитать до включения

- **[КРИТИЧНО] Общая сигнальная земля.** GND ESP8266 (питание по USB) и GND БП 12 В
  обязаны быть соединены — иначе PWM не имеет опоры и коммутация непредсказуема.
  Важно: «общая земля» относится к ESP и БП, но НЕ к минусам ламп — коммутируемые
  выводы ламп должны быть раздельными (см. схему).
- **[КРИТИЧНО] Предохранитель.** Slow-blow **T6.3–8 А** у выхода БП 12 В. Суммарно 68 Вт
  плюс пусковой бросок холодных нитей — это не настольная игрушка.
- **[КРИТИЧНО] Плата фонаря под напряжением.** В проверенной схеме общая плата
  фонаря — шина +12 В (постоянно «горячая»). Не допускать её контакта с чем-либо,
  связанным с GND: короткое примет предохранитель, но лучше без сюрпризов.
- **Полярность D4184.** На первом включении убедиться, что duty 0 = лампа ВЫКЛЮЧЕНА
  (нет инверсии). Все лампы после флэша/ребута обязаны быть выключены.

## Схема подключения (проверена на железе)

Особенность фонаря 2106: плата патронов конструктивно объединяет «минусы» всех ламп
(в машине коммутируется плюс, минус — кузов). Для low-side коммутации раздельными
должны быть именно коммутируемые выводы, поэтому схема **переполюсована** — лампе
накаливания полярность безразлична. Общей может быть только та сторона, которую
не коммутируем:

```
БП 12 В (+) ──[предохранитель T6.3–8 А]──┬── общая плата фонаря (общий ПЛЮС всех ламп)
                                         │        │ нить каждой лампы
                                         │   индивидуальный контакт патрона CHx
                                         │        │
                                         │   OUT− своего модуля D4184 (сток) — РАЗДЕЛЬНО
                                         │
                                         └── VIN+ модулей D4184
GND БП 12 В ── VIN− модулей D4184 ── GND ESP8266   (общая сигнальная земля)
ESP8266 GPIO4/5/15/13 ── PWM-входы модулей D4184
ESP8266 питание — отдельно, по USB
```

Если минусы ламп объединить до ключей (как штатно в фонаре), любой открытый ключ
замыкает петлю всех ламп сразу — все горят и диммируются вместе. Проверено.

## Карта каналов: секция → GPIO

| Idx | Key       | Название   | Мощность | GPIO   | Метка платы |
|-----|-----------|------------|----------|--------|-------------|
| CH0 | `stop`    | Стоп       | 21 Вт    | GPIO4  | D2          |
| CH1 | `reverse` | Задний ход | 21 Вт    | GPIO5  | D1          |
| CH2 | `turn`    | Поворот    | 21 Вт    | GPIO15 | D8          |
| CH3 | `marker`  | Габарит    | 5 Вт     | GPIO13 | D7          |

Плата стенда — **HW-364A** (ESP8266 + бортовой OLED). GPIO12/GPIO14 заняты I2C
дисплея, поэтому CH2 сидит на GPIO15 (D8), а не на GPIO12 из ТЗ.

**Каверза GPIO15:** это strapping-пин, при boot обязан быть LOW (pulldown уже стоит
на плате) — для low-side ключа это плюс: канал гарантированно закрыт при старте.
Но ничто внешнее не должно тянуть GPIO15 вверх при включении, иначе ESP не
загрузится. Затворный pulldown модуля D4184 условие выполняет; внешних pullup
на этой линии не добавлять.

Критерий выбора остальных пинов: не strapping (GPIO0/2), не GPIO16 (нет PWM);
при boot канал закрыт (pulldown затвора на модуле, если нет — добавить внешний).
Пины задаются в `src/config/config.cpp` (`CHANNEL_DEFS`); при импорте JSON поле
`gpio` игнорируется — перекоммутация «на лету» на силовом стенде опасна.

## OLED-дисплей и кнопка FLASH (плата HW-364A)

Бортовой SSD1306 128x64, I2C `0x3C`: **SDA = GPIO14 (D6), SCL = GPIO12 (D5)** —
разводка сверена с [peff74/esp8266_OLED_HW-364A](https://github.com/peff74/esp8266_OLED_HW-364A).
Кнопка **FLASH = GPIO0** переключает страницы (не держать при ресете — это вход
в bootloader). Обновление экрана — 2 раза/с; шрифт ASCII, подписи каналов латиницей.

| Страница | Содержимое |
|----------|------------|
| 1/3 Статус | IP, MAC, клиенты AP, heap, частота PWM, состояние капа, loop/s |
| 2/3 Каналы | по каналам: GPIO, слайдер %, текущий duty (виден soft-start) |
| 3/3 Калибровка | по каналам: gamma, min..max duty, soft_start_ms |

`loop/s` — частота главного цикла: просадки видны при HTTP-запросах и отрисовке
дисплея, полезно при оценке джиттера softPWM. Если дисплей не найден, стенд
работает без индикации (в Serial будет `OLED not found`).

## Сборка и прошивка

Arduino IDE + ESP8266 core. Библиотеки: **Adafruit SSD1306** (+ зависимости
Adafruit GFX, Adafruit BusIO) — только для OLED; PWM и web — стандартный core.
Открыть `SmartLada.ino`, плата NodeMCU 1.0 (HW-364A совместима), прошить.

## Использование

1. После старта все 4 канала = 0%, кап `max_duty_cap = 60%` активен.
2. Подключиться к Wi-Fi AP **SmartLada** (пароль `smartlada`), открыть **http://192.168.4.1**.
   Страница полностью автономна (без интернета/CDN).
3. 4 слайдера — независимая яркость каналов; 0% = гарантированный физический выкл.
4. Поля калибровки на канал: `gamma` (1.8–2.6), `min_duty`/`max_duty` (0–1023),
   `soft_start_ms` (0–1000) — кнопка «Применить».
5. Глобально: `pwm_freq_hz` (100–1000, дефолт 150 — искать потолок softPWM) и кап.
   «Снять кап» — отдельная кнопка с подтверждением; переход к полной яркости через soft-start.
6. **Export JSON** — снять текущий конфиг для переноса на ESP32-C6.
   **Import JSON** — вставить и применить сохранённый конфиг.

Маппинг яркости (порядок зафиксирован):
`slider% → gamma → [min_duty..max_duty] → min(duty, cap) → soft-start ramp → PWM`.

**Конфиг не сохраняется во флэш** (намеренно: force-safe старт с дефолтами).
Снятые числа калибровки сохранять через Export JSON.

## Замечание про джиттер softPWM

PWM на ESP8266 программный. При активных HTTP-запросах (движение слайдера, поллинг
статуса) возможен джиттер яркости — это был ожидаемый объект проверки.

**Результат прогона (2026-07-06):** джиттер глазом не наблюдался; разница между
100 и 1000 Гц неразличима — тепловая инерция нитей накаливания сглаживает всё.
Выбрано 100 Гц. Побочное наблюдение: импульсный БП «поёт» на частоте PWM
(магнитострикция при импульсной нагрузке) — норма, выше частота = выше и тише тон.

## Архитектура (слоями)

```
SmartLada.ino        тонкая обвязка: force-safe старт, setup/loop
src/channels/        [ПЕРЕНОСИТСЯ на C6]  чистая математика duty: gamma, min/max,
                     кап, soft-start. Ни одного include Arduino/ESP8266/Wi-Fi.
src/config/          [ПЕРЕНОСИТСЯ на C6]  структура настроек + JSON (чистый C++).
src/platform_pwm/    [ВЫБРАСЫВАЕТСЯ]      analogWrite* ESP8266; на C6 → LEDC.
src/web/             [ВЫБРАСЫВАЕТСЯ]      AP + HTTP + UI; на C6 → Zigbee.
src/display/         [ВЫБРАСЫВАЕТСЯ]      OLED SSD1306 + кнопка FLASH (HW-364A).
```

HTTP API: `GET /` (UI), `GET /config.json`, `POST /config` (тот же JSON),
`POST /set?ch=N&pct=P`, `GET /status` (MAC, heap, частота, кап, проценты).
